## 작업 요약
이전 배포 후 세션과 저장된 오케스트레이션 작업이 열리지 않던 문제를 DB 스키마 호환 계층으로 복구했습니다. 또한 왼쪽 사이드바의 최근 세션 클릭 시 같은 워크벤치 페이지 안에서도 즉시 해당 세션으로 전환되도록 바꾸고, 순차 검토 체인 모드에서는 혼동을 줄이기 위해 왼쪽 모델 선택 패널을 숨기도록 정리했습니다.

## 변경 사항
### 추가된 기능
- `workflowControlJson` 컬럼 자동 보정용 세션 스키마 호환 계층 추가

### 수정된 사항
- 세션 목록 API와 세션 상세 API가 `workflowControlJson` 미적용 DB에서 실패하던 문제 수정
- 세션 복제, 최종 결과 지정, 브랜치 삭제 등 세션 관련 후속 작업이 구형 DB에서도 동작하도록 보강
- 워크벤치가 쿼리스트링 변화(`?session=...`, `?project=...`, `?preset=...`)를 즉시 반영하도록 수정
- 왼쪽 최근 세션 클릭 후 별도 "열기" 버튼 없이 바로 해당 세션 상세 작업 내용이 열리도록 수정
- 순차 검토 체인 모드에서 왼쪽 모델 선택 패널과 모바일 모델 탭을 숨기도록 수정
- 생성된 워크플로우 route 파일의 불필요한 ESLint 경고 제거
