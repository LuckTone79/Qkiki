## 작업 요약
워크벤치의 모델 선택 구조를 확장해, 같은 AI 공급자 안에서 여러 모델을 동시에 선택해 병렬 결과를 받을 수 있게 수정했다. 또한 단일 결과 카드의 `이어서 질문`/`다른 모델로 검토` 분기 실행 시 공급자별 세부 모델을 직접 골라 후속 실행할 수 있도록 UI와 타깃 생성 로직을 함께 보완했다.

## 변경 사항
### 추가된 기능
- 병렬 비교 실행에서 같은 공급자의 여러 모델을 동시에 선택하는 기능 추가
- 결과 카드의 `이어서 질문` 및 `다른 모델로 검토` 패널에 공급자별 모델 다중 선택 UI 추가
- 기존 단일 모델 선택 상태를 새 다중 선택 구조로 안전하게 정규화하는 보정 로직 추가

### 수정된 사항
- 공급자 선택 상태를 `provider + model 1개` 구조에서 `provider + models[]` 구조로 변경
- 병렬 실행 대상 생성 로직을 공급자별 복수 모델을 평탄화하여 API로 전달하도록 수정
- 공급자 로드 시 이전 상태와 기본 모델을 함께 고려하는 초기화 로직 개선

### 제거/정리된 사항
- 공급자별 단일 `<select>` 기반 모델 선택 UI 제거
