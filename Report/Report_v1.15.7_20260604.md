# 작업 보고서
## 기본 정보
- 버전: `v1.15.7-20260604`
- 작업 일시: `2026-06-04`
- 이전 버전: `v1.15.6-20260603`
- 프로젝트: `Qkiki / Multi AI Workbench`

## 작업 목적
이전 단계에서 정리한 사용자 중심 개선 Top 10 가운데, 기존 기능과 충돌 위험이 낮고 체감 효과가 큰 UX 항목을 우선 구현했습니다. 이번 목표는 아래 흐름을 더 쉽게 만드는 것이었습니다.

- 첫 진입을 더 단순하게 만들기
- 초보자용 간단 모드와 고급 모드 분리
- 순차 검토 체인의 실행 계획을 실행 전에 보여주기
- 현재 작업 요약을 상단에서 바로 이해되게 만들기
- 결과 보드를 탐색 도구로 강화하기
- 최종 결과를 더 빨리 찾고 활용하게 만들기
- 세션 목록에서 이어서 작업하기 쉽게 만들기
- 모델 선택에 추천/신뢰 정보를 보강하기
- 사용량 안내를 더 직관적으로 만들기

## 이번 버전 반영 사항

### 1. 랜딩 페이지 진입 단순화
- 랜딩에 `빠르게 비교하기`, `단계별 개선하기` 두 개의 빠른 시작 카드가 보이도록 구성했습니다.
- 사용자가 로그인이나 가이드북으로 이동하기 전에, Qkiki를 어떤 방식으로 시작할 수 있는지 먼저 이해할 수 있게 했습니다.

### 2. 워크벤치 간단 모드 / 고급 모드 분리
- 워크벤치 상단에 `간단 모드`, `고급 모드` 토글을 추가했습니다.
- 간단 모드에서는 질문 입력과 실행에 집중하고, 추가 지시/첨부/반복 설정/조기 종료 같은 고급 제어는 숨깁니다.
- 고급 모드에서는 기존 상세 기능을 그대로 유지합니다.
- 세션 복원, 드래프트 복원, 프리셋 적용 시에도 현재 상태에 맞춰 간단/고급 모드가 올바르게 다시 계산되도록 보완했습니다.

### 3. 순차 검토 체인 실행 계획 미리보기
- 순차 모드에서 `실행 계획 미리보기` 영역을 추가했습니다.
- 총 몇 단계가 실행되는지, 반복 구간이 어떻게 펼쳐지는지, 각 단계가 어떤 모델과 source mode를 쓰는지 실행 전에 확인할 수 있습니다.
- 반복 블록 개수와 예상 확장 단계 수를 함께 보여줘 반복 설정을 더 쉽게 점검할 수 있습니다.

### 4. 현재 작업 요약 강화
- 워크벤치 상단에 `현재 작업 요약` 카드를 추가했습니다.
- 병렬 모드에서는 완료 결과 수와 진행 중 결과 수를 요약합니다.
- 순차 모드에서는 총 단계 수, 완료 단계 수, 현재 단계, 대기 단계 수를 바로 읽을 수 있게 했습니다.

### 5. 결과 보드 필터 / 검색 / 정렬 추가
- 결과 보드에 검색 입력, 필터, 정렬을 추가했습니다.
- 필터:
  - 전체 결과
  - 최종 결과만
  - 실패만
  - 메인 결과만
  - 분기 결과만
- 정렬:
  - 워크플로우 순서
  - 최신순
  - 오래된순
  - 실패 우선
- 검색:
  - 결과 본문
  - 모델명
  - 단계 키워드
  - action/source 관련 토큰

### 6. 최종 결과 빠른 보기 추가
- 결과 보드 상단에 `최종결과 빠른 보기` 영역을 추가했습니다.
- 최종 결과가 있으면 어떤 결과가 최종 선택인지 먼저 보여주고, 해당 카드로 이동하거나 바로 복사할 수 있게 했습니다.

### 7. 세션 목록 실행 요약 강화
- 세션 목록 API가 최신 실행 요약을 함께 내려주도록 확장했습니다.
- 세션 카드에서 아래 정보를 바로 볼 수 있게 했습니다.
  - 완료 / 실패 / 진행 중 / 대기 중 단계 수
  - 순차 실행 진행률
  - 최종 결과 선택 여부
- 다시 열기 전에 어디까지 진행됐는지 빠르게 파악할 수 있게 했습니다.

### 8. 모델 선택 추천 / 신뢰 안내 보강
- 공급자별 모델 선택 카드에 추천/특성 배지를 추가했습니다.
- 예:
  - 추천 시작점
  - 빠름
  - 균형
  - 정교함
  - 검토용
- `선택한 모델 그대로 실행하며, 자동 fallback 없이 결과를 보여줍니다.`라는 안내를 추가해 모델 신뢰성을 더 명확히 했습니다.

### 9. 사용량 안내 문구 보강
- 사용량 카드에 실행 계획과 반복/첨부가 실제 사용량에 영향을 줄 수 있다는 설명을 추가했습니다.
- 문구를 읽기 쉬운 한국어/영어 상태로 정리했습니다.

### 10. 간단 모드 payload 정리
- 간단 모드에서 화면상 숨긴 고급 설정이 실제 실행 payload에 남지 않도록 정리했습니다.
- 간단 모드 실행 시에는 고급 입력을 제거하고, 고급 모드 실행 시에는 기존 상세 값을 유지합니다.

## 상위 모델 추가 검토 반영
이번 패치는 바로 확정하지 않고, 상위 모델 리뷰 결과를 반영해 한 번 더 정리했습니다. 리뷰 후 반영한 핵심 수정은 아래와 같습니다.

- 간단 모드에서 숨긴 고급 설정이 실제 실행 payload에 남던 문제 수정
- 드래프트 복원 / 프리셋 적용 / 세션 재열기 경로에서 `builderExperience`가 어긋나던 문제 수정
- 결과 필터/검색이 새 세션이나 새 실행까지 남아 결과를 가리던 문제 수정
- 결과 검색이 단계 관련 키워드를 찾지 못하던 문제 수정

## 추가 구조 수정

### 1. Next 16 동적 라우트 충돌 정리
- `src/app/api/workbench/runs/[runId]/steps/[stepIndex]/route.ts`와
  `src/app/api/workbench/runs/[runId]/steps/[orderIndex]/branch-rerun/route.ts`가
  같은 깊이에서 서로 다른 slug 이름을 써서 Next 16 dev 환경에서 충돌 경고를 만들고 있었습니다.
- step stop 라우트도 실제로는 `orderIndex`를 받고 있었기 때문에, 라우트 경로를 `[orderIndex]`로 통일했습니다.
- 남아 있던 빈 `[stepIndex]` 디렉터리도 제거해 dev 서버 경고 원인을 정리했습니다.

### 2. 로컬 trial 검증 환경 정리
- 로컬 `.env`는 `file:./dev.db`를 가리키고 있었지만 현재 Prisma schema는 PostgreSQL만 허용하도록 바뀌어 있어서, trial/auth 경로가 실제 DB 접근 시 멈추는 구조였습니다.
- 검증용 dev 서버는 `POSTGRES_PRISMA_URL`을 주입한 상태로 다시 띄워, trial 시작과 워크벤치 진입을 직접 확인했습니다.
