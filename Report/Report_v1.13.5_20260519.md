# 작업 보고서

## 기본 정보
- **버전**: v1.13.5-20260519
- **작업 일시**: 2026-05-19
- **이전 버전**: v1.13.4-20260519
- **프로젝트명**: Qkiki 오케스트레이션 워크벤치

## 작업 요약
순차검토체인에서 첫 단계 모델 호출이 75초 타임아웃으로 실패하면 전체 반복 체인이 조기 중단되고, 이후 반복 확장 단계가 모두 건너뜀 처리되는 구조적 문제를 수정했다. 실패한 단계의 오류 텍스트를 다음 단계의 정상 입력처럼 전달하지 않도록 복구 입력 정책을 추가했고, 반복 설정이 있는 순차 체인이 마지막 성공 결과 또는 원본 입력을 기준으로 가능한 범위까지 진행되도록 개선했다.

## 변경 사항

### 추가/개선 기능
- OpenAI 고추론 모델별 최소 타임아웃을 추가했다.
  - gpt-5.5: 180초
  - gpt-5.4: 150초
  - gpt-5.4-mini: 90초
- timeout/rate-limit/transient provider 오류는 총 3회까지 같은 선택 모델로 재시도한다.
- 순차 체인에서 단일 단계 실패가 즉시 전체 중단으로 이어지지 않도록 했다.
- 이전 단계 출력이 없을 때는 실패 메시지가 아니라 원본 작업 기반 복구 입력을 다음 단계에 전달한다.
- `all_results` 소스는 완료된 결과를 우선 사용하고, 완료 결과가 없으면 원본 입력을 복구 소스로 사용한다.
- 반복 확장 체인은 첫 완료 결과 전 연속 2회 실패 또는 완료 결과 이후 연속 3회 실패 때만 조기 중단한다.
- 부분 완료 상태에서는 전역 오류 배너 대신 실패 카드 확인 안내를 표시한다.

### 설계 보정
- 사용자가 선택한 provider/model 의미가 바뀌지 않도록 순차 체인에서 provider fallback을 강제하지 않는다.
- 실패 result id를 다음 단계의 정상 previous result id로 연결하지 않는다.
- 실패한 단계도 결과 카드로 남겨 사용자가 어느 단계에서 어떤 provider/model이 실패했는지 추적할 수 있게 했다.

## AI 모델 3회 검토 반영
- **gpt-5.4 검토**: retry 상수가 실제 의미와 다르고 `all_results`가 실패 메시지를 다음 입력으로 오염시킬 수 있다고 지적했다. 총 3회 시도 상수와 완료 결과 우선 정책으로 반영했다.
- **gpt-5.2 검토**: 순차 체인에서 provider fallback을 강제하면 사용자가 선택한 모델 의미가 바뀐다고 지적했다. fallback 강제를 제거하고 같은 선택 모델의 재시도만 유지했다.
- **gpt-5.4-mini 검토**: 부분 완료 메시지가 실패 세부 원인을 숨길 수 있다고 지적했다. 실패 결과 카드를 확인하도록 완료 안내를 보정했다.

## 알려진 이슈 / 추후 작업
- 반복 구간 자체의 순서 확장은 기존 템플릿 순서 기반을 유지한다. 이번 작업은 provider 실패가 반복 체인 전체를 즉시 무효화하지 않도록 하는 안정성 개선에 집중했다.
