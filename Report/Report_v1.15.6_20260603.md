## 작업 요약
결과보드의 접기 UX를 다시 설계해 `결과배치(1열/2열)`와 접힘 상태를 완전히 분리했습니다. 이제 결과보드 헤더에서 전체 접기/전체 펼치기를 할 수 있고, 각 결과 카드 상단에서도 개별 접기/펼치기를 바로 수행할 수 있습니다. 카드가 펼쳐진 상태에서도 상단 버튼이 남아 있어 다시 접기 위해 본문 하단까지 내려갈 필요가 없습니다.

## 변경 사항
### 결과보드 동작
- 결과보드 헤더 우측에 `전체 접기`, `전체 펼치기` 버튼을 추가했습니다.
- 전체 접기/펼치기는 메인 워크플로우 결과와 후속 질문/분기 결과를 모두 함께 제어합니다.
- 결과배치 1열/2열 토글은 카드 배치만 담당하고 접힘 상태에는 더 이상 영향을 주지 않게 분리했습니다.

### 결과 카드 동작
- 각 결과 카드 상단 메타 영역 우측에 `접기` / `펼치기` 버튼을 배치했습니다.
- 카드 본문 하단에 있던 접기 버튼은 제거했습니다.
- 접힌 상태에서는 결과 본문 또는 오류 메시지의 첫 줄만 표시합니다.
- 새 결과가 스트리밍으로 추가되거나 세션을 다시 열어도 기본 상태는 전체 접힘으로 시작합니다.

### 상태 관리
- 결과 ID 기준의 접힘 상태 맵을 별도 UI 상태로 관리하도록 변경했습니다.
- 기존 상태는 유지하면서 새로 들어온 결과만 기본 접힘으로 추가되는 헬퍼를 분리했습니다.
- 공유 링크 결과 화면도 같은 방식으로 카드별 접기/펼치기를 지원하도록 맞췄습니다.

### 테스트
- 결과 접힘 상태 초기화, 상태 병합, 전체 접기/펼치기 헬퍼 테스트를 추가했습니다.

- 실제 브라우저 상에서 긴 결과 카드와 매우 많은 결과가 섞인 세션을 한번 더 시각적으로 확인하면 더 안전합니다.
