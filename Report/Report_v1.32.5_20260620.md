# Report v1.32.5-20260620

## 문제

모바일의 프로젝트 메뉴에서 프로젝트 카드 안 텍스트가 길어질 때 카드 폭이 자연스럽게 유지되지 않고, 일부 버튼과 세션 목록이 가로로 밀려 보이는 문제가 있었습니다.

사용자가 화면을 축소해서 보는 방식이 아니라, 다른 메뉴와 같은 폭 규칙을 유지한 상태에서 긴 텍스트만 줄바꿈 또는 생략 처리하는 쪽으로 정리할 필요가 있었습니다.

## 변경 사항

- `src/components/projects/ProjectsClient.tsx`
  - 프로젝트 카드 자체에 `overflow-hidden`을 적용해 카드 바깥으로 가로가 새지 않게 정리했습니다.
  - 프로젝트명은 최대 2줄까지 보이고 이후는 생략되도록 조정했습니다.
  - 설명과 업데이트 메타 텍스트는 모바일에서 자연스럽게 줄바꿈되도록 수정했습니다.
  - `열기` / `새 대화창` 버튼 영역을 모바일 기준 2열 고정 폭으로 정리해 카드 바깥으로 밀리지 않게 맞췄습니다.
  - 카드 내부 세션 제목은 `truncate` 1줄 대신 2줄까지 허용하고 이후 생략되도록 바꿨습니다.
  - 세션 메타 정보도 줄바꿈 가능하도록 조정했습니다.
- `VERSION`, `src/lib/version.ts`
  - 버전을 `v1.32.5-20260620`로 올렸습니다.
