# Report v1.35.0 - 2026-06-18

## 변경 요약

랜딩 페이지의 "병렬 비교" / "순차 검토 체인" 섹션에 가이드북처럼 제품 고유 특장점을
보여주는 시각화 추가.

## 배경

가이드북 페이지에는 워크벤치 mock 화면(모델 토글·결과 카드·체인 흐름)이 있어 기능을
직관적으로 전달하지만, 랜딩 페이지의 병렬 비교/순차 검토 체인 섹션은 텍스트 위주라
이 프로그램만의 강점이 잘 드러나지 않았음.

## 변경 내용 (`src/app/page.tsx`)

### 병렬 비교 섹션 — "하나의 작업 → 동시 다중 답변" 시각화
- 상단에 하나의 프롬프트(예: "신제품 출시 카피를 써줘") 입력 카드
- "4개 모델에 동시 전송 ↓" 표시(teal 강조)로 병렬 실행 직관화
- GPT/Claude/Gemini/Grok 4개 결과 카드를 나란히 배치
  - 모델별 컬러 배지 + 답변 스켈레톤 + **지연시간(⏱)·토큰 수** 메타데이터
- 캡션: "탭 4개 열 필요 없이 답변·속도·비용을 한 화면에서 비교" — 경쟁 도구 대비 강점 강조
- 기존 3개 feature 칩은 보조 설명으로 유지

### 순차 검토 체인 섹션 — "출력 → 입력 자동 연결" 흐름 시각화
- 4단계 카드(GPT 초안 → Grok 비판 → Gemini 개선 → Claude 최종)를 화살표로 연결
  - 데스크톱: 가로 화살표(→), 모바일: 세로 화살표(↓) 반응형
  - 각 단계에 **역할 배지**(초안 생성/비판/개선/최종 완성) 추가
  - 최종 단계는 teal 강조 + "최종" 배지로 부각
- 캡션: "각 단계의 출력이 다음 단계 입력으로 자동 전달 — 복사·붙여넣기 없이" — 핵심 차별점 강조

### 정리
- 미사용이 된 `routeExample` 상수 제거, `parallelCards`·`chainFlow` 상수 신규 추가
- 한/영 i18n 모두 대응
