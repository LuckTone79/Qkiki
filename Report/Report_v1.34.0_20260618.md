# Report v1.34.0 - 2026-06-18

## 변경 요약

랜딩 히어로 가이드북 버튼을 브랜드 강조색(teal)으로 변경하여 가시성 강화 + 미반영 원인 분석.

## 미반영 원인 분석

사용자가 "랜딩 가이드북 버튼이 안 보이고, 가이드 페이지 순서도 안 바뀌었다"고 재차 보고함.
조사 결과:

- **코드는 정상 반영**되어 있었음. v1.33.0(PR #25)에서 `page.tsx`(가이드북 CTA 행 이동)와
  `guide/page.tsx`(섹션 순서 변경)가 모두 올바르게 머지됨(main `a93d295`).
- rebase 충돌 해결 과정에서도 해당 변경은 보존됨(두 파일 직접 검증 완료).
- **실제 원인은 배포 미전파**: 라이브 사이트(`yapp.wideget.net`)가 #25 머지 직전 상태
  (#24 리브랜딩까지만)를 서빙 중이었음. Vercel은 main push 시 자동 배포되나 빌드 전파
  지연 + CDN/브라우저 캐시로 옛 화면이 보였음.
- 부가 문제: v1.33.0에서 시작하기/가이드북이 **둘 다 흰색**이라 시각적으로 구분되지 않음.

## 변경 내용

### `src/app/page.tsx` — 히어로 가이드북 버튼

- 흰색(`bg-white/95`) → **teal 강조색(`bg-teal-500`)** + 흰 글자로 변경
- 두 흑백 CTA(시작하기=흰색, 로그인=검정) 사이에서 확실히 구별되는 컬러 버튼
- 책 아이콘 추가로 가이드북임을 직관적으로 표시
- `shadow-lg` + `ring-teal-300/40`으로 입체감 부여, hover 시 `bg-teal-400`

### 가이드 페이지 (`src/app/guide/page.tsx`)

- 추가 변경 없음. v1.33.0의 섹션 순서가 이미 올바름:
  병렬 비교 → 순차 검토 체인 → 기능 한눈에 보기 → 단계별 사용 가이드 → FAQ
- 본 배포로 라이브에 정상 전파됨.

## 배포 후 확인

- 브라우저 강력 새로고침(Ctrl+Shift+R)으로 CDN/캐시 우회 필요할 수 있음.
