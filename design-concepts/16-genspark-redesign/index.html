Qkiki · UI Redesign Proposal

“대충 만든 AI 티”를 걷어내는
7가지 리디자인 제안

첨부해주신 Genspark 앱 스크린샷 4장에서 디자인 언어를 추출했습니다 — 초대형 라운드 컴포저, 라인 아이콘 타일 그리드, 흑백 필(pill) 버튼 체계, 바텀시트, 파란 포인트의 플랜 카드. 이 언어를 Qkiki의 실제 화면(워크벤치·모델 선택·결과 카드·요금)에 그대로 입힌 7개 시안을 목업으로 만들었습니다. 아래 목업은 전부 실제 Qkiki의 기능과 문구 기준입니다.

진단

지금 UI가 “AI가 대충 만든 느낌”을 주는 이유

핵심 문제 5가지

  • 이모지 아이콘 — 내비게이션이 🧪🗂️📄⚡👤 이모지라서 OS마다 다르게 렌더링되고, 즉시 ‘생성형 초안’ 인상을 줍니다. 레퍼런스는 전부 일관된 스트로크의 SVG 라인 아이콘입니다.
  • 폼(form) 같은 입력부 — 텍스트영역 2개 + 점선 첨부 박스 + 버튼 나열. 레퍼런스는 입력을 ‘하나의 큰 라운드 카드’로 묶고 부가 기능은 + 버튼 뒤로 숨깁니다.
  • 체크박스 관리 화면 같은 모델 선택 — 공급자별 체크박스 행 + 안내문. 제품의 핵심 매력(멀티 모델)이 가장 사무적으로 보입니다.
  • 작은 라운드·작은 버튼·테두리 과다 — rounded-md(6px) 위주라 화면이 잘게 쪼개져 보입니다. 레퍼런스는 18~28px 라운드와 여백으로 위계를 만듭니다.
  • 포인트 컬러 부재 — teal을 잉크색으로 리매핑해 전부 회색조. 레퍼런스는 흑백 기반에 파랑 딱 한 곳(선택/업그레이드)만 씁니다.

01

컴포저 허브 홈

워크벤치 첫 화면 · 임팩트 최대

레퍼런스 홈 화면의 직역입니다. 접속 직후 보이는 것은 단 하나 — 브랜드 마크와 “무엇이든 물어보고, 여러 AI로 비교해보세요” 컴포저 카드. 텍스트영역 2개와 점선 첨부 박스를 카드 하나로 통합합니다.

  • 이중 텍스트영역 → 단일 컴포저. 추가 지시·첨부는 왼쪽 + 원형 버튼의 시트로 이동
  • 병렬/순차 모드는 “Standard ▾” 자리의 필 드롭다운으로
  • 실행 버튼은 오른쪽 검정 원형 전송 버튼 하나로
  • 프로젝트·세션·프리셋·이미지 생성은 아래 도구 타일 그리드로 승격

02

데스크톱 아이콘 레일 셸

AppShell 교체 · 데스크톱

Qkiki는 데스크톱 사용이 많으므로 같은 언어를 데스크톱에도 적용합니다. 264px 텍스트 사이드바를 64px 아이콘 레일로 좁히고(호버 시 라벨 툴팁, 클릭 시 패널 확장), 본문은 중앙 정렬 컴포저가 주인공이 됩니다.

  • 이모지 7종 → 스트로크 1.8px 라인 아이콘 세트로 전면 교체
  • 최근 세션·프로젝트 목록은 레일 클릭 시 슬라이드 패널로
  • 인사말(“무엇을 비교해볼까요?”) + 최근 작업 퀵 필 제공

03

모델 브랜드 타일

ProviderSelectorRow 교체 · 웹+모바일 공통

멀티 모델은 Qkiki의 존재 이유인데, 지금은 관리자 설정처럼 보입니다. 레퍼런스의 도구 타일 문법을 빌려 공급자를 브랜드 타일로 만들고, 탭 한 번으로 켜고 끕니다. 선택된 타일은 검정 테두리 + 체크 배지. 웹(데스크톱)과 모바일에 동일한 타일 문법을 적용하고, 아이콘은 각 사의 공식 로고 마크(OpenAI 노트 · Anthropic A · Gemini 그라디언트 스파크 · xAI 마크)를 그대로 씁니다.

  • 체크박스 행 + 상태 문구 → 브랜드 타일 (모바일 2×2 · 데스크톱 4열 한 줄)
  • 아이콘은 임의 도형이 아닌 공식 로고 마크 — 앱 아이콘과 같은 배경색(검정/크림/화이트) 위에 정식 글리프
  • 세부 모델은 타일 아래 칩(pill) 목록에서 다중 선택
  • “Configured by administrator” 같은 시스템 문구는 비활성 타일의 잠금 아이콘으로 축약
  • 하단 고정 검정 실행 필에 선택 요약(“3개 모델로 비교 실행”)

04

4탭 내비 + 바텀시트

모바일 IA 재편

지금 모바일 하단 바에는 탭이 7개나 있어 라벨이 10px로 구겨집니다. 레퍼런스처럼 홈 · 세션 · 프로젝트 · 더보기 4탭으로 줄이고, 첨부·프리셋·가이드·계정 같은 보조 기능은 바텀시트로 옮깁니다.

  • 점선 첨부 박스 → 컴포저 + 버튼이 여는 카메라 / 사진 / 파일 시트 (레퍼런스 4번 이미지 문법)
  • 프리셋 불러오기·프로젝트에 저장·검토 체인 설정은 시트의 아이콘 행 리스트로
  • 가이드·계정·피드백·언어는 더보기 탭 안으로
  • 시트는 그래버 핸들 + 26px 상단 라운드, 배경 딤 42%

결과 카드 스트림

ResultCard 재설계 · 핵심 오브젝트

Qkiki의 핵심 오브젝트인 결과 카드를 레퍼런스의 제안 카드 문법으로 다시 그립니다. 원형 모델 아바타 + 회색 라운드 카드, 텍스트 버튼 나열 대신 하단 아이콘 액션 행(복사·분기·개선·요약·채택). 채택된 카드만 흰 배경 + 검정 테두리로 승격됩니다.

  • 모델명·상태 배지 → 아바타 + 이름 + 소요시간 한 줄 헤더

  • ‘최종 채택’은 검정 테두리 + 채택됨 배지로 시각적 승자 표시
  • 하단 고정 “나란히 비교” 필로 diff 뷰 진입

모노 필 컴포넌트 시스템

전 화면 공통 기반 · globals.css

1~5번 시안이 전부 이 기반 위에 섭니다. 레퍼런스의 버튼 문법은 명확합니다 — 검정 채움 필 = 주 행동 1개, 아웃라인 필 = 보조, 고스트 = 3차, 파랑 = 선택·업그레이드 상태 전용. 현재 globals.css의 “teal→회색 리매핑”을 걷어내고 이 4단 위계로 교체합니다.

  • 모서리 체계: 카드 20~26px · 버튼/입력 999px(필) · 타일 17px — rounded-md(6px) 퇴출
  • 중립색: Notion 웜그레이 → 레퍼런스의 쿨 뉴트럴(#f4f5f6 / #141517)
  • 포인트: #2F6BFF 파랑 1색, 선택 상태·업그레이드·인기 배지에만 사용
  • 세리프(Lora) 헤딩 제거 → 산세리프 800 웨이트 + -0.02em 자간으로 통일
