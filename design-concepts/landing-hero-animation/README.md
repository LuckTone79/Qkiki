# 랜딩 히어로 — AI 큐브 네트워크 움짤 (컨셉)

현재 랜딩페이지(`src/app/page.tsx`)의 히어로 배경은 Unsplash 정적 이미지
(`photo-1639762681485-074b7f938ba0`, 큐브들이 데이터로 연결된 형상)를 사용합니다.
이 폴더는 그 이미지를 **크고 작은 큐브(AI 노드)들과 데이터가 서로 유기적으로
움직이는 움짤**로 재해석한 컨셉 산출물입니다.

> ⚠️ 아직 사이트에는 적용하지 않았습니다. 미리보기/검토용 컨셉입니다.

## 파일

| 파일 | 설명 |
| --- | --- |
| `cube-network.html` | Canvas 애니메이션 본체. 브라우저로 바로 열면 실시간 루프 재생. |
| `cube-network.gif` | 위 애니메이션을 캡처한 800×450 무한 루프 GIF (4초, 64프레임). |
| `render.mjs` | HTML을 헤드리스 Chromium으로 프레임 캡처 후 GIF로 인코딩하는 스크립트. |

## 컨셉

- **노드**: 크기가 제각각인 아이소메트릭 큐브 16개 = 각각의 AI.
  사인 함수 기반으로 천천히 드리프트(유기적 부유)하며, 각자의 리듬으로
  티일/시안 빛을 내며 "사고"하는 펄스.
- **엣지**: 가까운 큐브끼리 연결되는 데이터 패브릭(연결선). 거리가 가까울수록
  더 밝게 표현.
- **데이터 펄스**: 연결선을 따라 흐르는 밝은 점 = AI 간 주고받는 데이터.
- **루프**: 모든 모션이 정수 주기의 주기함수라 시작/끝 프레임이 이어지는
  완전 무한 루프(이음새 없음).
- **팔레트**: 히어로 오버레이와 어울리는 딥 네이비 배경 + 브랜드 티일(teal-400)
  / 시안 액센트.

## 재생성 방법

```bash
# 1) 임시 작업 폴더에서 의존성 설치 (저장소에는 포함하지 않음)
mkdir -p /tmp/gifgen && cd /tmp/gifgen && npm init -y
npm install playwright-core gifenc

# 2) 렌더 (인자: HTML, 출력GIF, 프레임수, 프레임당 ms)
node /절대경로/render.mjs \
  /절대경로/cube-network.html \
  /절대경로/cube-network.gif 64 55
```

`render.mjs`의 Chromium 실행 경로(`executablePath`)는 환경에 맞게 조정하세요.

## 사이트 적용 시 (추후, 승인 후)

GIF보다는 `cube-network.html`의 `<canvas>` 로직을 React 컴포넌트로 옮겨
히어로 `<Image>` 자리에 배경 레이어로 까는 방식을 권장합니다(선명도·용량·
반응형 측면에서 유리). 적용은 요청 주시면 진행하겠습니다.
